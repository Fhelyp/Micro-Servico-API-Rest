Bem-vindo!

🤷‍♂️O que é isso? Esta é uma API interna da Omie. Atualmente, possui um endpoint que foi criado para armazenar os IDs das conversas abertas no Intercom.

Intercom Endpoint

📌 Este endpoint permite que você consulte, adicione ou exclua IDs da lista de conversas atuais.

  • 🔎 Consultando conversas abertas

    O método GET retorna uma string que armazena os IDs das conversas atuais, listando os IDs mais recentes primeiro. Conteúdos retornados:

    • conversas_abertas - string com os IDs separados por espaço (“ ”) como delimitador.
    • total_count - número total de conversas atuais (int).

    Exemplo de uso: GET https://omie.fhelypg.repl.co/intercom

    Você receberá uma resposta contendo um JSON com os dados solicitados:

    {
    "conversas_abertas": " id1 id2 id3",
    "total_count": 3
}

  • ➕ Adicionando ID(s)

    O método POST permite adicionar novos IDs, através da inclusão de query params:

    • id - parametro com um ou mais IDs à serem incluídos (obrigatório)

    Observação: Para adicionar mais de um ID, é preciso incluir um espaço em branco entre eles (em requisições curl, o espaço será encodado para “%20”).

    Exemplo de uso: POST https://omie.fhelypg.repl.co/intercom?id=AAAA BBBB

    Você receberá uma resposta contendo um JSON com a listagem atualizada:

    {
    "conversas_abertas": " id1 id2 id3 AAAA BBBB"
}
  • 🗑️ Excluindo Contatos

    Um DELETE também pode ser usado para excluir um ou mais IDs existentes, basta informa-lo(s) na URL, assim como fizemos no método POST.

    Exemplos:

    DELETE https://omie.fhelypg.repl.co/intercom?id=AAAA
DELETE https://omie.fhelypg.repl.co/intercom?id=id2%20id3

    Você também receberá uma resposta contendo um JSON com a listagem atualizada:

    {
    "conversas_abertas": " id1 BBBB",
}
